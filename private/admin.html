Router panels

Each Cerebrum panel is identified by its fixed IP. Assign which heads it controls, its screen layout, and arrange them to match the physical monitor wall.

Head snapshot filters (global)

Set which snapshots are permitted for each head. These apply to that head on every panel.

Auto-share snapshots

Continuously checks the selected cards and marks any unshared snapshot as shared, so new snapshots propagate across boards automatically.

Scheduled backup

Exports a target board's snapshots on a daily schedule, dated and retained for a set number of days.

Sync diagnostics

Investigate the boards' native card-to-card snapshot sync. "Check" reads each board's sync config, live status, and snapshot flag breakdown (shared / read-only / deleted). "Trigger" manually kicks the board's sync and reports the result. Read-only for config — nothing is changed except the manual trigger.

Widget geometry probe (test tool)

Tests whether a board accepts overlapping / fullscreen / off-canvas widget geometry that the native GUI blocks — the open question for the "blow a window up to fullscreen" idea. Pick a head and a widget, set it fullscreen (or a custom geometry), and watch what the board does. This is a LIVE change to the on-air head — use a spare or off-air head. Every change is reversible with “Restore”, and nothing here is saved to config.

Board API activity (diagnostics)

Live log of every request this server makes to the multiviewer boards, including connection failures with their underlying cause. Use "Test reach" on a card to probe it without changing anything.
